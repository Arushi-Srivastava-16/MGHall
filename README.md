## 🧠 What is CHG?

Most LLM safety tools are reactive — they check the *output* after generation. **CHG is proactive**: it monitors the LLM's *reasoning process step by step* and intervenes before a hallucination propagates.

CHG converts an LLM's chain-of-thought into a **Causal Reasoning Graph (CRG)** — a directed graph where each reasoning step is a node. A **Graph Attention Network (GAT)** then analyzes this graph to:

1. **Detect** which steps are logically incorrect
2. **Locate** the origin of the first error
3. **Classify** the error type (factual, logical, syntactic)
4. **Intervene** via RAG context injection before bad reasoning spreads

> **Key Insight**: An error at step 2 of a 6-step chain contaminates 100% of downstream steps. Early detection is everything.

## ✨ Features

| Feature | Description |
|---|---|
| 🔍 **Forensic Fingerprinting** | Identify whether text was generated by GPT-4, Gemini, or Llama using a 24-feature linguistic classifier |
| 📉 **Real-Time Risk Scoring** | `VulnerabilityGAT` predicts hallucination risk at each reasoning step *before* the response is complete |
| 🌲 **Dynamic Branching Viz** | Visualize "Alternative Paths" (Branch A vs B) when high risk is detected |
| 🛡️ **RAG Intervention** | Logic-aware retrieval injects corrective context directly into the LLM prompt |
| 🤝 **Multi-Model Consensus** | Runs the same query through GPT-4 + Gemini + Llama and finds where they disagree |
| 📊 **Interactive Dashboard** | Full React frontend for live demos, forensic analysis, and model management |

### Data Flow

```
LLM Output
    │
    ▼
Reasoning Chain → CRG Builder → Graph (395-dim node features)
                                   │
                     ┌─────────────┼─────────────┐
                     ▼             ▼              ▼
              Node Classifier  Origin Det.  Error Type
              (Correctness)   (First Error)  (4 classes)
                     │             │              │
                     └─────────────┴──────────────┘
                                   │
                            Risk Score → RAG Intervention
```

**Node Feature Vector (395 dimensions):**
- `384-dim` — SentenceTransformer text embedding (`all-MiniLM-L6-v2`)
- `5-dim` — Topology features (degree, centrality, path length)
- `6-dim` — Task features (domain one-hot, step position, chain length)

## 📈 Models & Performance

Three domain-specific **ConfidenceGatedGAT** models are trained and served:

### Math GNN
| Metric | Node Classification | Origin Detection |
|--------|:-------------------:|:----------------:|
| Accuracy | 90.32% | 98.39% |
| Precision | 93.51% | 91.18% |
| Recall | 94.59% | 96.88% |
| **F1 Score** | **94.08%** | **94.03%** |

### Code GNN
| Metric | Node Classification | Origin Detection |
|--------|:-------------------:|:----------------:|
| Accuracy | 92.63% | 99.89% |
| Precision | 95.87% | 98.31% |
| Recall | 94.14% | 100.00% |
| **F1 Score** | **95.00%** | **99.15%** |

### Model Specs
- **Architecture**: 3-layer GAT, 4 attention heads, 128 hidden dim
- **Parameters**: ~250K per model (~200KB on disk)
- **Inference Latency**: ~10ms per graph
- **Throughput**: ~100 graphs/second

## 🚀 Quick Start

### Prerequisites

| Requirement | Version |
|---|---|
| Python | 3.10+ |
| Node.js | 16+ |
| Gemini API Key | [Get one here](https://aistudio.google.com/app/apikey) |

### 1. Clone the Repository

```bash
git clone https://github.com/Arushi-Srivastava-16/MGHall.git
cd MGHall
```

### 2. Set Up Environment Variables

```bash
cp .env.example .env
# Edit .env and add your Gemini API key:
# GEMINI_API_KEY=your_key_here
```

### 3. Install Python Dependencies

```bash
python -m venv venv
source venv/bin/activate          # On Windows: venv\Scripts\activate
pip install -r requirements.txt
```

> **Note**: PyTorch Geometric requires matching CUDA versions. See the [PyG install guide](https://pytorch-geometric.readthedocs.io/en/latest/install/installation.html) if you hit issues.

### 4. Install Frontend Dependencies

```bash
cd frontend
npm install
cd ..
```

### 5. Run the Demo

```bash
chmod +x run_demo.sh
./run_demo.sh
```

This single command:
- ✅ Kills any processes on ports 8000 / 5173
- ✅ Activates the virtual environment
- ✅ Starts the **FastAPI backend** on `http://localhost:8000`
- ✅ Starts the **React frontend** on `http://localhost:5173`

Press `Ctrl+C` to stop both servers.

## 🖥️ Usage Guide

Once the demo is running, open your browser to `http://localhost:5173`.

### Chat Demo (`/chat`)
- Type any math, code, or medical reasoning query
- Click **"Math Error"** to simulate a weak model sending flawed reasoning
- Watch the **risk score** update in real-time as each step is analyzed
- Toggle **RAG Intervention** to see the system correct hallucinations before they finalize

### Forensic Tool (`/forensic`)
- Paste any piece of text
- The classifier predicts whether it was generated by **GPT-4**, **Gemini**, or **Llama**
- Returns 24 extracted linguistic features (hedge density, vocab richness, formatting style, etc.)

### Chains Visualizer (`/chains`)
- View the Causal Reasoning Graph (CRG) built from a query
- Nodes colored by correctness prediction
- Red nodes = hallucinated steps; origin node highlighted

### Consensus Analysis (`/consensus`)
- Submit a query to all 3 models simultaneously
- View where models agree vs disagree step-by-step
